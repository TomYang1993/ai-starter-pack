# Design quality

Default AI-built UI reads as templated: centered card, one shadow, a blue
primary button, evenly-spaced everything. The job is to make deliberate choices
instead, so the result looks designed rather than defaulted.

## Make a choice, then commit to it

- **Pick a point of view** before coding: what is this UI's mood (calm, dense,
  editorial, playful, utilitarian)? Let it drive every later decision.
- **Type carries the design.** Choose a deliberate type scale with real contrast
  between levels. Set line-height and measure (line length) for reading comfort.
  One or two families, used with intent, beat a pile of weights.
- **Space is a tool, not a gap.** Use a consistent spacing scale. Let some areas
  breathe and others sit tight — uniform padding everywhere reads as flat.
- **Color with restraint and one clear accent.** A neutral base plus a single
  confident accent looks more intentional than many competing colors. Check
  contrast for legibility.
- **Hierarchy first.** The eye should land on the most important thing first.
  Size, weight, color, and position all signal rank — use them on purpose.

## Avoid the tells of generic output

- Not everything centered; not one drop-shadow on every box.
- No default blue button unless blue is a real choice here.
- Align to a grid; don't let elements float at arbitrary offsets.
- Match the host framework's real, available styling primitives — don't invent
  classes or tokens that won't resolve.

## Verify

Before finishing, look at the result as a whole: is there a clear focal point,
consistent spacing rhythm, and readable type? If it looks like a default
template, change one structural thing — not just colors.
